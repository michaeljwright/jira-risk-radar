Risk Radar


Risk Description Threshold
Security Important to security i.e. Authentication related would be high 0.7
Dependency Relates to a high dependency on or impacts other features 0.2
Relevance High visibility to the end customer (i.e. Consumer Facing) or end-user and/or core functionality 0.4
Importance Highly important to the business in terms of reliability and risk to failure in compliance 0.5
Rework Relates to work that has been identified as an existing bug that wasn't caught previously 0.6

Rating Action Jira Label
>= 7.5 Requires a label added to the ticket, requires QA testing and upfront E2E testing + below. Q1
>= 6.5 Requires a label added to the ticket, requires QA testing for Prep env before release + below. Q2
>= 5.5 Requires a label added to the ticket, can proceed without QA testing but requires PO to do UAT + below. Q3
Below 5.5 Does not require a label, engineers can proceed without QA testing as long as code coverage passes, functionality has been checked with other engineers etc. N/A
